### `npm start`

開発モードでアプリを実行します。
http://localhost:3000](http://localhost:3000)を開くとブラウザで表示されます。

編集するとページがリロードされます。
また、コンソールにlintエラーが表示されます。

### `npm test`

対話型ウォッチ・モードでテスト・ランナーを起動します。
詳細は、[running tests](https://facebook.github.io/create-react-app/docs/running-tests) のセクションを参照してください。

### `npm run build`

本番用のアプリを `build` フォルダにビルドします。
本番モードでReactを正しくバンドルし、ビルドを最適化して最高のパフォーマンスを実現します。

ビルドは minified され、ファイル名にはハッシュが含まれます。
アプリをデプロイする準備ができました！

詳細は、[deployment](https://facebook.github.io/create-react-app/docs/deployment) のセクションを参照してください。

### `npm run eject`

**注：これは一方通行の操作です。一度 `eject` したら、もう戻れません！** 。

ビルドツールや設定の選択に満足できない場合は、いつでも `eject` することができます。このコマンドを実行すると、プロジェクトからビルドの依存関係を削除します。

その代わりに、すべての設定ファイルと依存関係のあるもの（webpack、Babel、ESLintなど）をプロジェクトにコピーします。eject`以外のすべてのコマンドはそのまま使えますが、コピーされたスクリプトを指すので、それらを微調整することができます。この時点では、あなたは自分でやるしかない。

eject`を使う必要はない。キュレーションされた機能セットは小規模から中規模のデプロイメントに適しており、この機能を使うことを義務に感じる必要はありません。しかし、もしあなたがその準備ができたときにカスタマイズできなければ、このツールは役に立たないと私たちは理解しています。
